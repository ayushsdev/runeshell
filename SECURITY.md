# Security Policy

## Supported Versions

Security fixes are prioritized for:

- The latest commit on the default branch
- The most recent tagged release

Older releases may not receive fixes.

## Reporting a Vulnerability

Please do not open public GitHub issues for security reports.

Send reports to: asharmadev314@gmail.com

Include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Any suggested remediation

## Response Process

- Initial triage response target: within 72 hours
- Confirmed issues are remediated as quickly as possible
- We will coordinate disclosure timing with the reporter when feasible

## Disclosure

After a fix is available, we may publish a summary in release notes or an
advisory so users can apply patches quickly.
